# Translation Workflow Audit — NovelClaw

Status: v2 stabilization branch
Branch: `mika/fix-translation-core-v2`

## Goal

Stabilize the current translation workflow before continuing batch translation.

This branch does **not** implement the full future multi-language v3 architecture. It fixes the current v2 contradictions first so future translation work has a reliable base.

## Current Problems Found

### 1. Conflicting dialogue quote rules

The previous prompt said CN -> TH should output straight quotes (`"..."`), while `style.md` and `format_spec.json` define v2 dialogue as `「...」`.

Decision for v2:

```text
Store translated dialogue as 「...」 in chapter JSON.
```

Future v3 can move display quotes into profile-driven rendering.

### 2. Validation config contradicted locked glossary

`香江` is locked as `เซียนเจียง`, but validation config previously treated `ฮ่องกง` as correct.

Decision:

```text
香江 -> เซียนเจียง
ฮ่องกง / เซียงเจียง -> wrong in this novel context
```

### 3. Locked glossary typo

`穴居人` was listed as `มนุษย์ถ้า`. It should be `มนุษย์ถ้ำ`.

Decision:

```text
穴居人 -> มนุษย์ถ้ำ
```

### 4. Generated glossary pipeline was referenced but missing

`glossary.yml` says it is generated by `tools/build_yaml.py`, but that tool was missing.

Decision:

```text
Add tools/build_yaml.py.
Edit Markdown glossary sources first, then regenerate glossary.yml.
```

### 5. Chapter JSON files are inconsistent

Some files use:

```json
{ "format": "v2" }
```

Others use:

```json
{ "schema_version": 2 }
```

Decision:

```text
Use schema_version=2 as the canonical v2 field.
```

`tools/normalize_chapter_schema.py` was added to normalize metadata without rewriting prose.

### 6. meta.md progress was stale

`meta.md` reported translation progress around chapter 122, while translated files exist through chapter 138.

Decision:

```text
Update last_translated=138 and next_chapter=139.
```

## Immediate Workflow

### Step 1 — Fix v2 core contradictions

Done in this branch:

- align `PROMPT.md` with current v2 format;
- fix `validation_config.json` for `香江` and `穴居人`;
- fix `locked.md` typo for `穴居人`;
- add `tools/build_yaml.py`;
- add `tools/normalize_chapter_schema.py`;
- update `meta.md` progress.

### Step 2 — Regenerate glossary YAML locally

Run:

```bash
python tools/build_yaml.py --novel global-descent
```

Then inspect:

```bash
git diff novels/global-descent/glossary/glossary.yml
```

Expected important changes:

```text
穴居人 -> มนุษย์ถ้ำ
```

### Step 3 — Dry-run chapter normalization

Run:

```bash
python tools/normalize_chapter_schema.py --novel global-descent --start 1 --end 138
```

Review output. If it looks correct:

```bash
python tools/normalize_chapter_schema.py --novel global-descent --start 1 --end 138 --write
```

Then inspect:

```bash
git diff novels/global-descent/chapters/
```

### Step 4 — Validate before continuing translation

At minimum:

```bash
python -m pytest tests/
python tools/translate.py 138 --dry-run
```

If project CLI is still unstable, fix CLI/package issues before large batch translation.

## Future v3 Direction

Do not implement v3 until v2 is stable.

Future design:

```json
{
  "schema_version": 3,
  "source_lang": "cn",
  "target_lang": "th",
  "profile": "cn-webnovel-to-th-reader"
}
```

Move language-specific formatting/validation into:

```text
config/translation_profiles.yml
```

V3 should support:

- source_lang / target_lang separation;
- profile-based validation;
- source normalization;
- translation / normalize / polish / reformat modes;
- Thai original mode without unnecessary translation.
